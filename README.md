# PENJELASAN CODE PYTHON

Pada bagian paling awal, terdapat baris import datetime import os. Secara konsep, ini bertujuan untuk memuat dua modul bawaan Python, yaitu datetime dan os. Modul datetime digunakan untuk mengambil waktu saat ini ketika transaksi dilakukan, sehingga struk pembayaran dapat mencantumkan waktu yang akurat. Sementara itu, modul os digunakan untuk berinteraksi dengan sistem operasi, khususnya untuk menjalankan perintah membersihkan layar terminal. Perlu diperhatikan bahwa secara penulisan yang benar, kedua import ini seharusnya dipisah menjadi dua baris (import datetime dan import os), karena jika digabung seperti itu akan menimbulkan error sintaks.

Selanjutnya, program mendefinisikan sebuah kelas bernama SistemTol. Kelas ini berfungsi sebagai kerangka utama dari seluruh sistem, yang di dalamnya berisi data dan fungsi-fungsi yang saling berkaitan. Dengan menggunakan kelas, program menjadi lebih terstruktur dan mudah dikembangkan.

Di dalam kelas tersebut terdapat metode __init__, yaitu konstruktor yang akan otomatis dijalankan ketika objek dari kelas dibuat. Pada bagian ini, program menginisialisasi dua atribut penting. Atribut pertama adalah self.tarif_dasar, yaitu sebuah dictionary yang menyimpan data tarif untuk setiap golongan kendaraan. Struktur dictionary ini memungkinkan penyimpanan data yang kompleks, di mana setiap golongan memiliki informasi berupa tipe kendaraan, harga dasar, dan persentase diskon. Misalnya, golongan “1” merepresentasikan mobil pribadi dengan tarif 50.000 dan diskon 50%, sedangkan golongan “3” untuk truk besar mendapatkan diskon 25%. Atribut kedua adalah self.log_transaksi, yaitu sebuah list kosong yang digunakan untuk menyimpan nilai transaksi yang telah berhasil dilakukan. Setiap transaksi yang sukses akan ditambahkan ke dalam list ini sehingga nantinya bisa digunakan untuk menghitung total pendapatan.

Berikutnya terdapat metode bersihkan_layar. Metode ini berfungsi untuk membersihkan tampilan terminal agar tidak penuh dengan output sebelumnya. Di dalamnya terdapat perintah os.system('cls' if os.name == 'nt' else 'clear'), yang berarti program akan menjalankan perintah cls jika dijalankan di sistem operasi Windows, dan clear jika dijalankan di Linux atau macOS. Dengan cara ini, program menjadi lebih rapi dan nyaman digunakan.

Metode selanjutnya adalah cetak_struk, yang bertugas untuk menampilkan hasil transaksi dalam bentuk struk pembayaran. Metode ini menerima beberapa parameter seperti golongan kendaraan, jumlah uang yang dibayar, kembalian, nama gerbang, harga asli, dan besar diskon. Di dalam metode ini, program terlebih dahulu mengambil waktu saat ini menggunakan datetime.datetime.now() lalu memformatnya menjadi bentuk yang mudah dibaca menggunakan strftime. Setelah itu, program mencetak informasi transaksi secara terstruktur, dimulai dari nama gerbang, waktu transaksi, jenis kendaraan, harga sebelum diskon, jumlah diskon, hingga total pembayaran. Selain itu, juga ditampilkan uang yang dibayarkan dan kembalian. Penggunaan "="*35 dan "-"*35 berfungsi sebagai garis pembatas agar tampilan struk terlihat rapi seperti struk asli.

Bagian inti dari program terdapat pada metode mulai_simulasi. Metode ini pertama-tama memanggil fungsi bersihkan_layar untuk membersihkan tampilan, kemudian menampilkan judul program. Setelah itu, pengguna diminta memasukkan nama gerbang tol yang akan digunakan selama simulasi. Nilai yang dimasukkan disimpan dalam variabel lokasi dan akan digunakan di berbagai bagian program, seperti tampilan menu dan struk.

Program kemudian masuk ke dalam perulangan while True, yang berarti menu akan terus ditampilkan sampai pengguna memilih untuk keluar. Di dalam perulangan ini, program menampilkan empat pilihan menu, yaitu memasukkan kendaraan, melihat laporan pendapatan, membersihkan layar, dan keluar dari program. Pilihan pengguna disimpan dalam variabel pilihan.

Jika pengguna memilih opsi pertama, program akan menampilkan daftar tarif dan diskon untuk setiap golongan kendaraan. Data ini diambil dari dictionary tarif_dasar menggunakan perulangan for. Program juga menampilkan informasi diskon jika ada, dengan mengubah nilai diskon menjadi persen. Setelah itu, pengguna diminta memasukkan golongan kendaraan yang diinginkan. Program kemudian memeriksa apakah input tersebut valid dengan mengecek apakah ada di dalam dictionary.

Jika golongan valid, program mengambil harga dasar dan menghitung potongan diskon dengan mengalikan harga dengan persentase diskon. Hasil diskon ini kemudian dikurangkan dari harga asli untuk mendapatkan harga akhir yang harus dibayar. Program lalu meminta pengguna memasukkan jumlah uang atau saldo. Pada bagian ini digunakan blok try-except untuk menangani kemungkinan kesalahan input, misalnya jika pengguna memasukkan teks вместо angka.

Jika saldo yang dimasukkan cukup untuk membayar tarif, program menghitung kembalian dan menambahkan nilai transaksi ke dalam log_transaksi. Setelah itu, metode cetak_struk dipanggil untuk menampilkan bukti pembayaran. Namun, jika saldo tidak mencukupi, program akan menampilkan pesan bahwa saldo kurang beserta jumlah kekurangannya.

Jika pengguna memilih menu kedua, program akan menampilkan laporan pendapatan. Di sini, program menghitung total pendapatan dengan menjumlahkan seluruh nilai dalam log_transaksi menggunakan fungsi sum. Selain itu, jumlah kendaraan dihitung dari panjang list tersebut menggunakan len. Informasi ini memberikan gambaran sederhana mengenai total transaksi yang telah dilakukan.

Menu ketiga berfungsi untuk membersihkan layar dengan memanggil metode bersihkan_layar, sedangkan menu keempat digunakan untuk keluar dari program. Ketika opsi keluar dipilih, perulangan akan dihentikan menggunakan perintah break, dan program akan menampilkan pesan penutup.

Pada bagian paling akhir terdapat blok if __name__ == "__main__":. Bagian ini berfungsi untuk memastikan bahwa kode hanya akan dijalankan jika file tersebut dijalankan secara langsung, bukan diimpor sebagai modul di file lain. Di dalamnya, program membuat objek dari kelas SistemTol dan kemudian memanggil metode mulai_simulasi untuk memulai program.
